## PR title

Test flow: five-image layout and four-image ordered comprehension scoring

## Summary (2-3 bullets)
- Portrait questions with five images use a 2×2 grid plus a centered fifth option, aligned with existing multi-image styling.
- Two-step ordered comprehension on exactly four images now scores full success in two correct taps, partial when the task completes on the third tap with the required order on taps two and three, and failure otherwise (including duplicate wrong first picks).

## numbered main changes
1. Added `images-container--five-up` layout and matching CSS so five options render as four plus one centered row on portrait mobile (comprehension and expression).
2. Extended comprehension tile styling for `data-count="5"` to match the tighter six-image treatment where applicable.
3. Implemented a dedicated click sequence for ordered two-step answers when there are four photos: two-click `first→second` success, three-click partial only when clicks two and three are `expFirst→expSecond`, duplicate wrong first tap fails immediately, and other three-click endings fail.

## numbered notes
1. Ordered scoring changes apply only when the question has exactly four images and a two-step `->` answer; other image counts keep the previous rescue behavior.
