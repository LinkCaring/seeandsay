## PR title

Incremental segment blob vault (5.8)

## Summary (2-3 bullets)

- Protects incremental expression audio from call/interrupt losses by vaulting segment blobs in IndexedDB (with memory cache) before enqueue, emergency snapshot on mic kill, and background/finish backfill without blocking Continue.
- Recorder uses 2s MediaRecorder timeslices and `emergencySnapshot()`; vault entries purge on successful register and at test end; `clientInfo.segmentVault` reports pending, skipped, and interrupt snapshot counts at finalize.
- Bumped app to **5.8**; legacy full-session mode unchanged.

## numbered main changes

1. Added `segmentBlobVault.js`: IndexedDB key `testId|questionNumber`, memory fallback when IDB unavailable, `put`/`get`/`remove`/`listPendingKeys`/`purgeTest`, telemetry helpers.
2. `expressionSegmentRecorder.js`: `recorder.start(2000)` timeslice capture; `emergencySnapshot(questionNumber)` builds partial blob without full cleanup.
3. `test.js`: vault before enqueue, vault read when `stopSegment` returns null, `flushVaultedExpressionSegments` on mic recovery, `online`, and finish; interrupt path snapshots to vault; queue `onQuestionCompleted` removes vault entry.
4. `expressionSegmentUploadQueue.js`: optional `onQuestionCompleted(key)` after successful register.
5. `testSessionFinish.js`: flush vault before finish retry burst; `getExpressionSegmentFinalizeClientInfo` for finalize opts; `purgeSegmentBlobVaultForTest` after metadata upload.
6. `apiToMongo.js`: `collectClientInfo` includes `segmentVault` from finalize opts; `MILI_APP_VERSION` **5.8**.
7. `index.html`: load `segmentBlobVault.js` before upload queue; footer **5.8** in `app.js`.
8. `frontend_demo/README.md`: documented segment vault behavior under incremental recording.

## numbered notes

1. Vault cannot invent audio when no chunks were captured (Q33–34 never started); server may still show `processing_failed`.
2. Q36–37 Azure audio with missing scored rows is a separate server race — out of scope for this change.
3. QA: call on expression Q during incremental — partial audio should vault, flush on mic recovery/finish, and `segmentVault.pending` should be empty after successful uploads and end-of-test purge.
