## PR title

iOS expression mic check AudioContext resume (5.7)

## Summary (2-3 bullets)

- Fixed expression sound-check gate on some iPhones where the level meter stayed flat despite granted mic permission by creating `AudioContext` before `getUserMedia` and calling `resume()` after wiring the analyser.
- Analyser loop runs only when `audioCtx.state === "running"`; tap-again message when resume fails instead of a silent flat meter.
- Bumped app to 5.7; added `test.mic.check.audioNotReady` i18n strings.

## numbered main changes

1. `testMicIntro.js`: create `AudioContext` synchronously on Start tap (before `await getUserMedia`); `ensureAudioContextRunning()` with `resume()` after analyser connect; gate `tick` on `running` state.
2. `i18n.js`: Hebrew/English `test.mic.check.audioNotReady` for failed audio context start.
3. Bumped footer and `MILI_APP_VERSION` to **5.7** in `app.js` and `apiToMongo.js`.

## numbered notes

1. Does not address zombie mic streams (live track, no input) or quiet-mic threshold tuning — retest on the originally affected iPhone after deploy.
2. If resume fails, parent can tap Start again (fresh user gesture).
3. QA: iPhone Chrome — login → comprehension → expression mic gate; meter should move within ~1s of normal speech.
