## PR title

Incremental expression audio pipeline and scoring reliability

## Summary (2-3 bullets)

- Delivered end-to-end incremental expression recording with login mode selection, segment upload APIs, and synchronized docs across root, frontend, and backend READMEs (including interrupt recovery and results routing).
- Fixed last-question race and incomplete AI progress by enqueue-before-finish, backend retry/fallback scoring, frozen expression timer during incremental upload drain, expression countdown UI tick when armed, and summary UI that completes on terminal `done`.
- Hardened incremental finalize with queue drain, mic release on success, full-screen segment-interrupt recovery (mic-gated restart at timer arm if mic was off earlier), and session score counters reconciled from deduped results at finish.

## numbered main changes

1. Added login recording mode selection in `welcome` flow and persisted `expressionAudioMode` to local storage for downstream behavior.
2. Added frontend incremental pipeline modules: `expressionSegmentRecorder` and `expressionSegmentUploadQueue`, and integrated enqueue-on-traffic-submit without blocking question advance.
3. Added backend segment upload support with new endpoints (`prepareSegmentUpload`, `expressionSegment`) and Azure segment blob path helper.
4. Added Mongo helpers for draft shell tests and idempotent per-question segment upsert, then merged draft shells on `addTestToUser`.
5. Updated `addTestToUser` to allow incremental metadata-only finalize (no full session audio required in incremental mode) while keeping legacy full-audio flow.
6. Added expression recording safety so segment blobs are bound to their originating expression question and cannot be reused across different expression items.
7. Updated `README.md`, `frontend_demo/README.md`, and `backend/docs/README.md` to document login mode selection and exact `legacy` vs `incremental` runtime behavior and API flows.
8. Updated `testScoring.js` to enqueue incremental segment uploads before question advance/finish and await queue idle (60s) when the session is about to complete.
9. Updated `testSessionFinish.js` to reconcile `correct`/`partly`/`wrong` from deduped `questionResults` at incremental finish and extend segment queue drain to 60s.
10. Added backend `_ensure_incremental_expression_rows_complete` with `retrying_missing` phase, configurable retries, and `processing_failed` fallback (`ai_score` 1) before impression build.
11. Set incremental finalize progress to full `total_questions` only after all rows exist; trigger results SMS on incremental finalize completion; summary UI requires `done` with `processed >= total` and gates impression until fully complete.
12. Freeze expression 20s eval countdown during incremental segment upload and finish queue drain; mark question end at traffic submit so timestamps stay capped while uploads complete.
13. Bumped app footer and `MILI_APP_VERSION` client metadata to 5.3.
14. Fixed expression eval countdown UI by re-running the tick interval when `expressionEvalArmed` becomes true (popup/timestamp timeouts unchanged).
15. Added `SessionRecorder.releaseCaptureStream()` (no MP3 encode) and incremental-only `releaseIncrementalCaptureResources()` on successful incremental finish.
16. Added segment health monitor (`track.onended`, inactive recorder while armed) with visibility re-check and 2s poll; suppresses during pause, clapping, and upload freeze.
17. Extended segment upload queue with per-question state, `cancelPendingForQuestion`, and `waitForQuestionIdle`; restart blocked after server register.
18. Added incremental-only interrupt modal and per-question restart flow; legacy `recordingInterruptedBanner` and finish paths unchanged.
19. Bumped app footer and `MILI_APP_VERSION` client metadata to 5.4.
20. Gate incremental interrupt restart on live `getUserMedia` probe; show mic-disabled hint until the microphone is available again.
21. During incremental segment interrupt, freeze the expression countdown, hide test navbar and question UI, and show a full-screen mic-recovery screen only.
22. Remove incremental interrupt dismiss unless the segment was already registered on the server; re-record is mandatory when restart is still allowed.
23. On incremental expression timer arm, probe mic and segment start health; if the mic was off during prompt or between questions, show interrupt/restart at arm time without modal during prompt.
24. Fixed stale `localStorage.page === "results"` without URL token routing to the results error screen on normal app open; only adjusts the `page` route key, not test resume data.
25. Synced `README.md`, `frontend_demo/README.md`, `backend/docs/README.md`, `BACKEND_MODULE_MAP.md`, `SMS_RESULTS.md`, `frontend_demo/docs/STRUCTURE.md`, and `frontend_demo/js/app/README.md` with all May 28 behavior above.
