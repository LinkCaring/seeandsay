## PR title

Incremental expression audio pipeline

## Summary (2-3 bullets)

- Added an end-to-end incremental expression path: per-question segment recorder, async upload queue, and backend segment ingestion/scoring APIs.
- Added login-level recording mode control (`legacy` vs `incremental`) and wired mode metadata into client info and finish behavior.
- Added incremental hardening for queue-drain-at-finish, terminal impression finalize, strict expression-recording-to-question mapping, and synchronized README documentation across root/frontend/backend docs.

## numbered main changes

1. Added login recording mode selection in `welcome` flow and persisted `expressionAudioMode` to local storage for downstream behavior.
2. Added frontend incremental pipeline modules: `expressionSegmentRecorder` and `expressionSegmentUploadQueue`, and integrated enqueue-on-traffic-submit without blocking question advance.
3. Added backend segment upload support with new endpoints (`prepareSegmentUpload`, `expressionSegment`) and Azure segment blob path helper.
4. Added Mongo helpers for draft shell tests and idempotent per-question segment upsert, then merged draft shells on `addTestToUser`.
5. Updated `addTestToUser` to allow incremental metadata-only finalize (no full session audio required in incremental mode) while keeping legacy full-audio flow.
6. Added expression recording safety so segment blobs are bound to their originating expression question and cannot be reused across different expression items.
7. Added incremental hardening: finish waits briefly for segment queue drain, segment phase moves from scoring to building summary, and finalize backfills missing rows from uploaded segments before impression.
8. Updated `README.md`, `frontend_demo/README.md`, and `backend/docs/README.md` to document login mode selection and exact `legacy` vs `incremental` runtime behavior and API flows.

## numbered notes

1. Backend must be restarted with the latest code (prefer `--reload`) so incremental finalize and phase transitions are applied to new tests.
