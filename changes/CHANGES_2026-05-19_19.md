## PR title

Fresh testId per game; fix stale run progress and incomplete-finish confirm

## Summary (2-3 bullets)

- Each new game clears prior run state and assigns a new `testId`, so AI scoring and the summary only cover the current run.
- Finish waits for MP3 conversion (scaled timeout, preparing UI) before Azure upload; failed uploads can be retried without re-taking the test.
- Incomplete-finish confirmation and mid-test resume behave correctly (no stale AI, no wiped progress).

## numbered main changes

1. `apiToMongo.js`: `beginNewTestSessionIdentity()` also clears in-progress run keys (`questionResults`, scores, recording marks); keeps age/login flags.
2. `test.js`: `resetFreshTestRunReactState()` on fresh start; `requestCompleteSessionOrConfirm()` centralizes incomplete-summary gating.
3. Resume: app **Continue** clears `forceFreshStartAfterMicCheck`; fresh-start effect bails when `hasInProgressTestState()`; age unlock uses `ensurePendingTestId()` only.
4. New-game starts set `forceFreshStartAfterMicCheck` + `sessionCompleted` false before `beginNew` so a second test cannot reuse prior `questionResults` / AI in React state.
5. `recording.js`: `whenFinalBlobReady()` promise + `getConversionWaitMs()` (1–5 min from active capture); `test.js` preparing screen, no upload without audio, Retry on failure.
6. Prior: new UUID + cleared `lastCompletedTestId` / `expressionAiResult` on new game; 12:30 active-recording cap unchanged.
7. Footer label in `app.js` updated from `version 4` to `version 4.1`.
